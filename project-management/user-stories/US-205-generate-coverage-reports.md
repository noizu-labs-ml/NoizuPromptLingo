## Description

As a QA engineer, I want to verify test coverage for all extended loading functionality.

This ensures code quality standards are met, identifies untested code paths, and provides visibility into testing completeness for the NPL loading extension feature.

---

## Acceptance Criteria

- [ ] **AC-5.1**: All new code has >= 80% line coverage
- [ ] **AC-5.2**: Coverage reports generated by `mise run test-coverage` command
- [ ] **AC-5.3**: Critical paths have 100% coverage:
  - Expression parser
  - Section resolver
  - Priority filter
- [ ] **AC-5.4**: Report shows gaps and uncovered branches with clear identification
- [ ] **AC-5.5**: No regression in existing test coverage (all previous tests still pass)

---

## Technical Notes

- Use pytest-cov for coverage measurement
- Coverage must be measured for all new modules:
  - `src/npl_mcp/npl/parser.py`
  - `src/npl_mcp/npl/resolver.py`
  - `src/npl_mcp/npl/filters.py`
  - `src/npl_mcp/npl/layout.py`
  - `src/npl_mcp/npl/loader.py`
- Critical paths requiring 100% coverage:
  - Parser: expression tokenization and validation
  - Resolver: component lookup and resolution
  - Filter: priority boundary handling
- HTML coverage report provides visual gap analysis

## Test Coverage Requirements

- **Line coverage**: >= 80% for all new code

## Commands

```bash
# Generate coverage report
mise run test-coverage

# Generate HTML coverage report with visual gaps
mise run test-html

# View HTML report
open htmlcov/index.html

# Quick test status check
mise run test-status

# See failing tests
mise run test-errors
```
