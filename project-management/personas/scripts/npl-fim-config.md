# npl-fim-config - Persona

**Type**: Script
**Category**: Configuration & Query Tool
**Version**: 1.0.0

## Overview

`npl-fim-config` is a configuration and query tool for the NPL-FIM (Noizu Prompt Lingo - Fill-In-the-Middle) agent. It provides natural language querying of visualization tool recommendations, manages tool-task compatibility matrices, maintains local override files for project-specific customizations, and delegates to `npl-load` for metadata loading.

## Purpose & Use Cases

- **Tool Discovery**: Query which visualization library best fits a specific requirement using natural language
- **Compatibility Reference**: Display tool-task compatibility matrices showing which libraries support which visualization types
- **Project Customization**: Create and maintain local override files for project-specific tool configurations and style guides
- **Workflow Integration**: Generate `npl-load` commands to load appropriate FIM metadata for current tasks
- **Preference Management**: Access preferred solution recommendations organized by visualization use case

## Key Features

✅ **Natural Language Queries** - Ask "org chart for React" and get ranked tool recommendations (mermaid, d3_js, go_js)
✅ **Compatibility Matrix** - Comprehensive tool-task mapping across 8+ visualization categories
✅ **Local Overrides** - Project-level customization via `.npl/meta/fim/*.local.md` files
✅ **Intelligent Delegation** - Derives appropriate `npl-load` commands from item paths (e.g., `d3_js.data-visualization`)
✅ **Hierarchical Metadata** - Respects `$NPL_META` environment variable with fallback search paths
✅ **Artifact Path Management** - Configurable output directory via `$NPL_FIM_ARTIFACTS`

## Usage

```bash
# Natural language query
npl-fim-config --query "interactive network graph for React"

# Display compatibility matrix
npl-fim-config --table

# Get preferred solutions for use case
npl-fim-config network-graphs --preferred-solution

# Load FIM metadata for tool+use-case
npl-fim-config d3_js.data-visualization --load

# Edit local override
npl-fim-config solution.mermaid.diagram --local --edit
```

The tool supports two primary workflows: **querying** for tool recommendations based on natural language descriptions, and **managing** local project overrides for FIM metadata. Query operations return ranked results with contextual recommendations (e.g., "Works well with React" for d3_js). The `--load` delegation automatically constructs hierarchical metadata paths and invokes `npl-load`.

## Integration Points

- **Triggered by**: NPL-FIM agent when selecting visualization approach, developers querying tool options, build scripts requiring artifact paths
- **Feeds to**: `npl-load` for metadata loading, FIM agent for tool selection, local override system for customization persistence
- **Complements**: `npl-load` (metadata loader), `npl-session` (work tracking), NPL-FIM agent (visualization generation)

## Parameters / Configuration

- **`--query`, `-q`** - Natural language description of visualization requirements (e.g., "3D graphics with WebGL")
- **`--table`** - Display full tool-task compatibility matrix, optionally filtered by item
- **`--artifact-path`** - Print configured artifact output directory (`$NPL_FIM_ARTIFACTS` or `./artifacts`)
- **`--preferred-solution`** - Show ordered list of preferred tools for specified use case
- **`--load [ENTRY]`** - Delegate to `npl-load` with derived metadata paths from item or explicit entries
- **`--local --edit`** - Open project-specific override file in `$EDITOR` (creates from template if missing)
- **`--local --patch --prompt`** - Append content to local override (preserves existing)
- **`--local --replace --prompt`** - Replace entire local override file

## Success Criteria

- Query returns 1-5 ranked tools matching natural language requirements with relevance scores
- Compatibility matrix displays complete tool coverage for specified use cases
- Local overrides persist in `.npl/meta/fim/` hierarchy and load correctly via `npl-load`
- Delegation to `npl-load` constructs valid metadata paths (e.g., `fim.solution.d3_js`, `fim.use-case.data-visualization`)
- Artifact path configuration respects environment variables and provides sensible defaults

## Limitations & Constraints

- Compatibility matrix is statically defined in script (not dynamically discovered from metadata)
- Natural language query uses keyword matching rather than semantic understanding
- Local overrides follow naming convention `*.local.md` and must be manually synchronized with upstream metadata changes
- Delegation to `npl-load` requires that tool be available in `$PATH`

## Related Utilities

- **npl-load** - Resource loader that consumes FIM metadata paths generated by this tool
- **npl-fim** - Visualization agent that uses tool recommendations from queries
- **npl-persona** - Persona management system that may consume FIM artifacts for visualization tasks
