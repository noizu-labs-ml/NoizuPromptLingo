# Persona: Product Manager

**ID**: P-002
**Created**: 2026-02-02
**Updated**: 2026-02-02

## Demographics

- **Role**: Product Manager / Project Lead
- **Tech Savvy**: Low to Medium (non-technical stakeholder)
- **Primary Interface**: Web UI dashboard and visual tools

## Context

A product manager or project lead who oversees AI agent work and development progress within the NoizuPromptLingo ecosystem. Does not write code but needs visibility into what agents and developers are producing. Reviews artifacts (documents, code, screenshots), approves work, and provides strategic direction through comments and task prioritization. Strongly prefers visual interfaces and dashboards over command-line tools.

## Goals

1. Track progress on features and tasks at a glance
2. Review artifacts (documents, screenshots, code) without technical setup
3. Provide feedback and approval on deliverables
4. Communicate priorities and blockers to the team
5. Maintain audit trail of decisions and approvals

## Pain Points

1. No visibility into what AI agents are doing
2. Difficulty reviewing technical artifacts without context
3. Cannot easily provide feedback on specific parts of documents
4. Task status updates require chasing down developers
5. Screenshots and visual artifacts scattered across tools

## Behaviors

- Checks dashboard for session and task queue status daily
- Reviews artifacts shared in chat rooms
- Adds inline comments and overlay annotations during reviews
- Reassigns or reprioritizes tasks based on business needs
- Uses notifications to stay informed of mentions and updates

## Quotes

> "I need to see what the team accomplished this sprint without reading code."

> "Can I just click on a screenshot and add a comment where the issue is?"

> "I should be notified when an artifact is ready for my review."
