# npl-onboarding

User onboarding specialist that creates personalized learning experiences through progressive disclosure, performance demonstration, and measurable success tracking.

## Purpose

Bridges NPL's sophisticated capabilities and mainstream adoption by providing role-based learning paths, interactive tutorials, and visible performance metrics. Transforms invisible 15-40% improvements into tangible user experiences.

See [Detailed Documentation](./npl-onboarding.detailed.md) for complete reference.

## Capabilities

- Create progressive learning paths customized by role
- Generate interactive tutorials with hands-on practice
- Demonstrate performance improvements through baseline comparison
- Track user progress with retention checkpoints
- Establish habit formation through daily practice
- Measure adoption success with quantified metrics

## Quick Reference

```bash
# Start individual onboarding
@npl-onboarding start --role="developer" --pace="moderate" --goal="code-review-enhancement"

# Set up team program
@npl-onboarding team-setup --size=15 --domain="data-science" --duration="4weeks"

# Check progress
@npl-onboarding progress --user="current" --metrics="comprehensive" --recommendations=true

# Generate success story
@npl-onboarding success-story --period="last-30days" --format="executive-summary"
```

See [Command Reference](./npl-onboarding.detailed.md#command-reference) for all parameters.

## Learning Path Phases

| Phase | Days | Focus |
|:------|:-----|:------|
| Foundation | 1-3 | Basic syntax, first success |
| Core Features | 4-7 | Agents, formatting, pumps |
| Advanced | 8-14 | Multi-agent, templates |
| Mastery | Ongoing | Optimization, mentoring |

See [Learning Path Architecture](./npl-onboarding.detailed.md#learning-path-architecture) for phase details.

## Integration

```bash
# Measure effectiveness
@npl-onboarding baseline --capture=true && @npl-performance measure --comparison="pre-post-onboarding"

# Accessible onboarding
@npl-accessibility review --onboarding-flow && @npl-onboarding adapt --accessibility="wcag-aa"

# Research-driven optimization
@npl-user-researcher survey --phase="onboarding" --week=1 && @npl-onboarding optimize --based-on="user-feedback"
```

See [Integration Patterns](./npl-onboarding.detailed.md#integration-patterns) for workflow details.
